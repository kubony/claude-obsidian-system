# Meeting Summarizer - 회의록 요약 스킬

녹음 파일이나 회의 텍스트를 분석하여 구조화된 요약 노트를 생성하는 스킬입니다.

## 사용 시점

- "회의록 정리해줘"
- "녹음 내용 요약해줘"
- "미팅 노트 만들어줘"

## 기능

- **자동 분석**: Claude Sonnet으로 텍스트 분석
- **참석자 추출**: 대화에서 언급된 인물 파악
- **인물사전 참조**: 04_Networking/00_인물사전/에서 참석자 배경 정보 조회
- **날짜/시간 추출**: 파일명 또는 내용에서 자동 파악
- **주요 주제**: 핵심 논의 사항 정리
- **액션 아이템**: 할 일, 결정 사항 추출
- **요약 생성**: 전체 내용을 간결하게 요약 (인물 배경 지식 활용)

## 출력 형식

```markdown
# [날짜] [주제] - [참석자]

## 참석자
- 이름1
- 이름2

## 회의 일시
YYYY-MM-DD HH:MM

## 주요 주제
- 주제1
- 주제2

## 논의 내용

### 주제1
- 세부 내용

### 주제2
- 세부 내용

## 액션 아이템
- [ ] 담당자: 할 일 설명
- [ ] 담당자: 할 일 설명

## 결정 사항
- 결정1
- 결정2

## 참고사항
- 기타 메모
```

## 인물사전 활용

**recording-processor 에이전트가 제공하는 경우:**
- 파일명에서 추출한 참석자 이름으로 인물사전 검색
- `04_Networking/00_인물사전/*{이름}*.md` 패턴으로 조회
- 발견된 인물 정보를 컨텍스트로 활용

**인물사전 정보 활용 방법:**
1. **소속/직함 정확성**: 인물사전의 정보로 정확한 조직명, 직책 파악
2. **관계 맥락**: 이전 미팅 내용, 프로젝트 연관성 확인
3. **참석자 식별**: 동명이인 구분, 호칭(님, 대표 등) 정리
4. **맥락 있는 요약**: 배경 지식을 바탕으로 더 풍부한 요약 작성

**인물사전이 없는 경우:**
- 내용 분석으로 추론
- 파일명과 대화 내용에서 최대한 정보 추출

## 볼트 소유자 식별

**중요:** 미팅 녹음/문서에는 높은 확률로 볼트 소유자가 참석자로 포함됩니다.

**볼트 소유자 정보:**
- **이름**: 서인근 (Inkeun Seo)
- **인물사전 파일**: `04_Networking/00_인물사전/서인근_본인.md`

**STT/OCR 오인식 패턴:**
음성 인식이나 OCR에서 다음과 같이 잘못 인식될 수 있습니다:
- 서인호, 서민호, 서민근, 서인군, 서인건
- 인근, 인근씨, 인근님
- Simon, Inkeun (영문)

**처리 규칙:**
1. 위 유사 이름이 참석자로 나타나면 **서인근**으로 교정
2. 미팅에서 "나", "저", "제가" 등 1인칭 화자가 명확히 볼트 소유자인 경우 서인근으로 표기
3. 소속/직함이 필요하면 `서인근_본인.md` 파일 참조 (현재: 1인 창업가/솔로프리너)
4. 참석자 목록에서 서인근은 별도 표기 불필요 (본인이므로)

## 사용 방법

이 스킬은 **에이전트 또는 Claude에 의해 호출**됩니다.
직접 사용보다는 `recording-processor` 에이전트를 통해 사용하세요.

텍스트 파일을 읽고 Claude가 분석하여 구조화된 마크다운을 생성합니다.
에이전트가 인물사전 정보를 제공하면 이를 참고하여 더 정확한 요약을 생성합니다.
